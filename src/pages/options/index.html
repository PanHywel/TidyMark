    ☁️ GitHub 云同步

    配置个人访问令牌与仓库信息，备份将自动保存到默认路径

    将在目标仓库创建备份文件；路径示例：tidymark/backups/tidymark-backup.json 或 tidymark/backups/tidymark-bookmarks.html。

    说明：同步为单向备份，仅将本地书签备份到仓库；不会从仓库还原到浏览器。

    尚未同步

    GitHub 操作指南
    1. 创建或准备仓库：可在 GitHub 主页右上角 + → New repository 新建仓库，默认分支通常为 main。
    2. 生成个人访问令牌（PAT）：进入个人设置 → Developer settings → Personal access tokens，创建令牌并至少勾选 repo 权限；复制令牌保存到 GitHub Token 字段。
    3. 填写同步配置：Owner（用户名或组织名）、Repo（仓库名）、备份格式（JSON/HTML）。可启用“同时上传两种格式”以在仓库生成两份文件。
    4. 验证与导入：点击“一键同步到 GitHub”后，到仓库查看文件是否更新。若选择或启用 HTML，同步的文件可在 Chrome 书签管理器导入（Chrome：书签管理器 → 右上角“⋮” → 导入书签）。

    🔧 使用整理功能

    1 点击“⚡ 自动整理”生成预览；如启用 AI，将进行二次优化。
    2 点击“🤖 AI 全量归类”推理新分类并预览，确认后执行移动。
    3 在确认弹窗中查看摘要与分类列表，确认后开始整理。

    🤖 AI 全量归类提示

    AI 全量归类会基于书签内容推理分类并生成预览，您可在预览中调整分类后确认执行移动。请先在「AI 分类助手」中配置服务与模型。

    建议在执行前先备份书签；该过程可能耗时，取决于书签数量与网络情况。

    📥 导入书签

    如需恢复或导入书签，请使用浏览器自带的导入功能：

    1 打开 Chrome 设置 → 书签 → 导入书签和设置
    2 选择要导入的书签文件
    3 确认导入后，重新运行 TidyMark 整理功能

    💾 备份建议

    为了保护您的书签数据，建议：

    • 定期使用浏览器的导出功能备份书签
    • 开启 Chrome 同步功能
    • 在整理前先导出当前书签
